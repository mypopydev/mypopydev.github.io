# 专题总览

`series/` 目录专门放“成组阅读才有价值”的内容。每个专题都应该有自己的 slug 目录和一个 `index.md` 作为专题首页；专题内部可以保留自己的阅读顺序、上一篇/下一篇导航和配套资源，而不需要和站点里的其他专题或独立文章硬混在一起。

当前已经落位的专题如下：

- [《Agentic Engineering Patterns》中文译文专题](./agentic-engineering-patterns/index.md)

如果你以后还要加第二个专题，建议直接新增一个兄弟目录，例如 `series/another-topic/`，并保持同样结构：专题首页、正文文件、专题内图片目录、专题说明页。这样以后无论是继续做翻译专题、读书专题，还是工具实践专题，都不用推倒现在的站点结构。
